# Change-Aware Coverage

## Problem

Traditional coverage metrics (62% line coverage) are not actionable for an AI agent. After modifying code, an agent needs three answers:

1. **Which methods I touched have no test coverage?**
2. **How risky are the uncovered gaps?** (persistence > view helpers)
3. **What downstream code depends on what I changed?**

The data sources to answer all three already exist — Clover XML, the attribute graph, and git diff. This plan wires them together into a single script.

## Data Sources

| Source | What it provides | How to access |
|--------|-----------------|---------------|
| `git diff` | Changed files and line ranges | `git diff --name-only <base>..HEAD` |
| `var/coverage/clover.xml` | Per-file, per-method, per-line coverage | Generated by `check:coverage` (PCOV + PHPUnit) |
| Attribute graph (JSON) | Layer, edges, dependents per node | `list:attributes -- --format=json` |

### Clover XML structure (relevant subset)

```xml
<package name="ZeroToProd\Thryds\Controllers">
  <file name="/app/src/Controllers/RegisterController.php">
    <class name="RegisterController" namespace="ZeroToProd\Thryds\Controllers">
      <metrics methods="2" coveredmethods="1" statements="10" coveredstatements="7"/>
    </class>
    <line num="15" type="method" name="__invoke" count="3"/>
    <line num="18" type="stmt" count="3"/>
    <line num="22" type="stmt" count="0"/>  <!-- uncovered -->
  </file>
</package>
```

### Attribute graph structure (relevant subset)

```json
{
  "_index": { "controllers": ["RegisterController", ...], "core": ["Database", ...] },
  "_dependents": { "RegisterController": [{"from": "Route", "rel": "handles"}] },
  "nodes": {
    "ZeroToProd\\Thryds\\Controllers\\RegisterController": {
      "file": "src/Controllers/RegisterController.php",
      "layer": "controllers",
      "methods": ["__invoke"]
    }
  }
}
```

## Design

### New artifacts

| File | Purpose |
|------|---------|
| `scripts/check-change-coverage.php` | The script |
| `scripts/change-coverage-config.yaml` | Configuration (layer risk weights, thresholds) |

### Config: `change-coverage-config.yaml`

```yaml
# Base ref for git diff. "auto" picks merge-base with main, falls back to HEAD~1.
base_ref: auto

# Clover XML location (must exist — run check:coverage first).
clover_file: var/coverage/clover.xml

# Risk weight per layer. Higher = more important to cover.
# Uncovered methods in higher-weight layers sort first in output.
layer_weights:
  core: 10
  tables: 9
  schema: 8
  queries: 7
  controllers: 6
  requests: 5
  routing: 4
  viewmodels: 3
  views: 2
  ui: 1

# Fail (exit 1) if any uncovered method is in a layer at or above this weight.
fail_threshold: 7
```

### Algorithm

```
1. Resolve base ref
   - If "auto": git merge-base main HEAD, fallback HEAD~1
   - Otherwise: use literal value

2. Get changed files
   - git diff --name-only --diff-filter=ACMR <base>..HEAD
   - Filter to src/**/*.php (exclude tests, scripts, config)

3. Parse Clover XML
   - For each changed file, find matching <file> node
   - Extract per-method coverage: { name, covered: bool, statements, coveredStatements }
   - A method is "uncovered" if coveredstatements == 0

4. Load attribute graph
   - Run: php scripts/list-attributes.php --format=json --sections=nodes,_dependents,_index
   - Map each changed file → FQCN → layer, dependents

5. Build risk report
   For each changed file:
   a. Look up its node in the attribute graph → layer, dependents
   b. Look up its methods in Clover XML → covered/uncovered
   c. Score: layer_weight × uncovered_method_count
   d. List dependents (from _dependents) as "impact radius"

6. Output JSON to stdout
   {
     "passed": true|false,
     "base_ref": "abc1234",
     "changed_files": 5,
     "summary": {
       "covered_methods": 8,
       "uncovered_methods": 2,
       "uncovered_risk_score": 14
     },
     "uncovered": [
       {
         "file": "src/Queries/CreateUser.php",
         "class": "CreateUser",
         "layer": "queries",
         "risk_weight": 7,
         "method": "execute",
         "dependents": ["RegisterController"],
         "statements": { "total": 12, "covered": 0 }
       }
     ],
     "covered": [
       {
         "file": "src/Controllers/RegisterController.php",
         "class": "RegisterController",
         "layer": "controllers",
         "method": "__invoke",
         "statements": { "total": 10, "covered": 7 }
       }
     ],
     "violations": ["Uncovered method CreateUser::execute in layer 'queries' (weight 7 >= threshold 7)"]
   }

7. Exit code
   - 0 if no uncovered methods in layers >= fail_threshold
   - 1 otherwise
```

### Stderr output (human-readable)

```
╔══════════════════════════════════════╗
║     Check: Change Coverage           ║
╚══════════════════════════════════════╝

Base: abc1234 (main merge-base)

┌─ Uncovered (risk-ordered)
│
│  [queries w:7]  CreateUser::execute        12 stmts, 0 covered
│                 ↳ dependents: RegisterController
│
│  [views w:2]    RegisterView::variables     4 stmts, 0 covered
│                 ↳ dependents: (none)

┌─ Covered
│
│  [controllers w:6]  RegisterController::__invoke  10/10 stmts
│  [core w:10]        Database::run                  8/8 stmts

Summary: 2 uncovered methods, risk score 9
Result: FAIL — uncovered methods in layers >= threshold (7)
```

### Integration into check:all

Add to `scripts/checks-config.yaml`:

```yaml
check:change-coverage:
  command: "php scripts/check-change-coverage.php"
```

This runs alongside existing checks. On a clean main branch with no diff, it reports 0 changed files and passes.

### Edge cases

| Case | Behavior |
|------|----------|
| No changed files (clean branch) | Pass with empty report |
| Changed file not in Clover XML | Report as "no coverage data" — treat as uncovered |
| Changed file not in attribute graph | Use layer "unknown" with weight 0 (won't fail) |
| Clover XML missing | Fail with violation: "Run check:coverage first to generate clover.xml" |
| New file (added, not modified) | Included — `--diff-filter=ACMR` catches additions |
| Deleted file | Excluded — `--diff-filter=ACMR` skips deletions |
| Non-src file changed (test, script) | Filtered out in step 2 |

### File-to-node mapping

Clover XML uses absolute container paths (`/app/src/...`). The attribute graph uses project-relative paths (`src/...`). Normalize both to project-relative before joining.

## What this does NOT do

- **Mutation testing** — valuable but slow; separate concern for a future script.
- **Assertion density** — requires PHPUnit extension; out of scope.
- **Line-level diff intersection** — method-level granularity is sufficient. Knowing "you changed CreateUser::execute and it has 0 coverage" is actionable. Knowing exactly which lines within it are uncovered adds noise without changing the action (write a test for the method).

## Implementation order

1. Create `scripts/change-coverage-config.yaml`
2. Create `scripts/check-change-coverage.php`
   - Steps 1-2 first (git diff parsing), test manually
   - Add step 3 (Clover XML parsing)
   - Add step 4 (attribute graph loading)
   - Wire together for steps 5-7
3. Add `check:change-coverage` entry to `scripts/checks-config.yaml`
4. Add `check:change-coverage` composer script to `composer.json`
5. Run `./run check:all` to verify integration
