# ForbidHardcodedCodeReferencesInCommentsRector

Flags code references in comments that are not reachable by AST refactoring tools, keeping comments evergreen across renames and restructures.

**Category:** Code Quality
**Mode:** `warn`
**Auto-fix:** No

## Rationale

Comments that name specific classes, attributes, methods, or constants become stale when the referenced code is renamed or removed. AST-aware refactoring tools (Rector, IDE rename) can update structured references (`@see`, `@param` types, `@return` types) but cannot reach prose text buried in comment strings. The rule: if a reference **can** be updated via AST refactoring, it is fine; everything else must be flagged.

## What It Detects

| Pattern | Example | Why it's fragile |
|---------|---------|-----------------|
| Attribute syntax | `#[RouteOperation]` | Rename attribute → comment is stale |
| Static member access | `RouteRegistrar::register()` | Rename class or method → stale |
| Arrow method call | `->dispatch()` | Rename method → stale |
| Fully-qualified class name | `\App\Routes\RouteRegistrar` | Move or rename class → stale |
| Code refs in `@param`/`@return` descriptions | `@param string $x Calls Foo::bar()` | Type portion is AST-reachable, description is not |

## What It Does NOT Flag

| Pattern | Why it's safe |
|---------|--------------|
| `@see ClassName::method` | Structural PHPDoc tag — AST-reachable |
| `{@see ClassName::method}` | Inline structural tag — AST-reachable |
| `@link ClassName::method` | Structural tag — AST-reachable |
| `@param Type $var` (type portion) | PHPDoc type — AST-reachable |
| `@return Type` (type portion) | PHPDoc type — AST-reachable |
| `@var Type` | PHPDoc type — AST-reachable |
| `@throws Type` | PHPDoc type — AST-reachable |
| `@phpstan-*`, `@psalm-*` | Static analysis annotations — AST-reachable |
| `@template`, `@extends`, `@implements` | Generics — AST-reachable |
| `@method`, `@property`, `@mixin` | Virtual declarations — AST-reachable |
| Rector TODO comments | Generated by tooling, not hand-written prose |

## Transformation

### In `warn` mode

```
// TODO: [ForbidHardcodedCodeReferencesInCommentsRector] '#[RouteOperation]' is not AST-reachable — use @see/@link or describe behavior without naming implementations. See: utils/rector/docs/ForbidHardcodedCodeReferencesInCommentsRector.md
```

## Configuration

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `mode` | `string` | `'warn'` | Only `warn` is supported — adds a TODO comment |

## Example

### Before

```php
class Router
{
    /** Dispatch to the handler declared on the #[RouteOperation]. */
    public function dispatch(): void {}
}
```

### After

```php
class Router
{
    // TODO: [ForbidHardcodedCodeReferencesInCommentsRector] '#[RouteOperation]' is not AST-reachable — use @see/@link or describe behavior without naming implementations.
    /** Dispatch to the handler declared on the #[RouteOperation]. */
    public function dispatch(): void {}
}
```

## Resolution

When you see the TODO comment:

1. **Remove the implementation detail** — rewrite the comment to describe *behavior*, not *mechanism*
2. **Or use a structural tag** — move the reference to `@see` or `{@see}` where AST tools can maintain it

### Rewrite examples

| Before | After |
|--------|-------|
| `// Calls RouteRegistrar::register()` | `// Binds declared routes to the HTTP kernel` |
| `/** Dispatch on the #[RouteOperation]. */` | `/** Dispatch to the declared route handler. */` |
| `// Then ->dispatch() runs` | `// Then the request is dispatched` |
